### Codebase conventions
- Bun runtime only, no build step, no dist/. TS runs directly.
- No `__dirname` — use `import.meta.url` / `import.meta.dir`.
- Tests: `*.test.ts` next to source, `bun test`.
- Fake executor pattern: `GhExecutor` interface in `lib/gh-cli.ts` — inject fake in tests.
- `mkdtempSync(join(tmpdir(), ...))` for temp dirs in tests.
- Biome for lint/format: `bun run check` / `bun run fix`.

### GitHub architecture (3 transports)
1. `gh-cli` — `issue_*`, `gh_run` (gh CLI subprocess)
2. `external-mcp` — PR comments/replies/merge-state (external GitHub MCP, not toolkit)
3. `journal-only` — PR watch state (`pr_*` tools, no GitHub network)

### Key policy decisions
- `gh_run pr merge` → DENY
- `issue_status` / `dryRun:true` → NO journal append by default
- `dryRun:false` apply → journal append (one entry)
- PR watch QA: fake/golden only, no external MCP required
- Live QA target: `minjun0219/agent-toolkit-test-target-repo` only
- GitHub config: no tokens/API URLs in toolkit config; `gh` owns auth/repo/GHE
